Atividade Avaliativa Controle de Versão

Controle de Versão:

    É um método para controlar todas as alterações que vão ser feitas no código de um projeto. Ele permite que várias versões do código sejam armazenadas, sendo desnecessário que o fique guardando várias cópias no computador. Ele funciona de maneira simples, os arquivos ficam armazenados dentro de um repositório, onde fica guardado todas versões dele. Um dos sistemas mais usados é o Git Hub.

Controle de Mudanças:

    Na gestão de projetos, controle de mudanças é um meio em que todas as mudanças que serão realizadas no projeto, ou código, sejam solicitadas para análise da equipe, que vão ter a oportunidade de aprovarem ou não. É basicamente um controle para qualquer alteração, evitando erros e também deixando a equipe sempre informada quando qualquer mudança for feita.

Integração Contínua:

    Integração contínua é uma prática de DevOps, na qual é criado um repositório central, que vai ser armazenado o código do projeto, e nele todas as mudanças no código vão ser juntadas. Isso permite que testes sejam feitos regularmente, com objetivo de evitar possíveis erros. O tempo de produção também vai ser reduzido, descartando o cenário de antes, onde os programadores trabalhavam sozinhos e só depois juntavam seus códigos em um.
